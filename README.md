# Gemini 文献综述生成助手

## 项目简介

Gemini 文献综述生成助手是一个基于 Google Gemini AI 的工具，能够自动处理学术文献并生成高质量的文献综述或定性研究论文。该工具提供友好的 Web 界面，让研究人员能够轻松地从 PDF 文献中提取信息，生成总结，并最终组织成结构化的学术文稿。

## 功能特点

- **PDF文献处理**：自动识别和处理PDF格式的学术文献
- **引用信息提取**：关联文献与引用信息
- **智能文献总结**：使用Gemini AI为每篇文献生成结构化总结
- **自动生成文稿**：基于总结生成完整的文献综述或定性研究论文
- **Web界面操作**：直观的三步式操作流程
- **缓存机制**：避免重复处理相同文献，提高效率

## 安装与配置

### 环境要求

- Python 3.12+
- 网络连接（用于API调用）
- Google Gemini API密钥

### 安装步骤

1. 克隆或下载项目到本地

2. 安装依赖：

   ```bash
   pip3.12 install -r requirements.txt
   ```

3. 配置 API 密钥：
   - 在 config.yaml 文件的 `llm` 部分配置 `api_key`

4. 自定义配置文件：
   - 编辑 config.yaml 文件以调整各项参数
   - 确保文件路径指向正确位置

### 配置文件说明

```yaml
llm:
  model_name: "gemini-2.0-flash"                                      # 用于文献总结的模型
  final_draft_model_name: "gemini-2.0-flash-thinking-exp-01-21"       # (可选) 用于最终生成的模型，可能需要更强能力
  request_timeout: 300                                                # API 请求超时时间 (秒)
  api_key: "API_KEY填到这里"                                           # 替换为您的 Gemini API 密钥
  temperature: 1.2                                                    # 0.0 表示确定性输出，2.0 表示高度随机
  max_retries: 3                                                      # 最大重试次数

paths:
  pdf_dir: "PDF_Files/" 
  txt_dir: "TXT_Files/"
  cache_file: "cache/summaries_cache.json"
  summary_output_file: "Results_Files/文献总结.json"
  final_output_file: "Results_Files/最终结果.txt"

prompts: # 文件名相对于 txt_dir
  research_theme: "研究主题.txt"
  summary_prompt: "文献总结提示词.txt"
  review_final_prompt: "文献综述撰写提示词.txt"
  qualitative_final_prompt: "定性研究论文撰写提示词.txt"
  references_list_file: "参考文献列表.txt"

# 缓存设置
cache:
  enabled: true
```

## 详细使用指南

### 准备工作

1. **准备PDF文件**：
   - 将所有需要处理的PDF文献放置在 PDF_Files 目录下
   - 文件名应避免特殊字符

2. **准备引用信息**：
   - 在 参考文献列表.txt 中添加文献引用信息
   - 每行一个引用，格式应与学术引用格式一致
   - 确保引用信息与PDF文件名能够关联匹配（即文件名需要包含于对应的参考文献列表）
   - 例如：
      - `参考文献：` [1] 本刊编辑部. 九位院士、校长谈“人工智能赋能高等教育”——“‘人工智能赋能教育’中国工程科技论坛”会议综述[J/OL]. 西安交通大学学报(社会科学版), 2023(3).
      - `文件名：` 人工智能赋能教育’中国工程科技论坛”会议综述

3. **设置研究主题**：
   - 在 研究主题.txt 中写入您的研究主题
   - 确保主题清晰明确，能够指导文献总结和最终文稿的生成

4. **检查提示词文件**：
   - 确保 TXT_Files 目录中包含所有必要的提示词文件：
     - `文献总结提示词.txt`
     - `文献综述撰写提示词.txt`
     - `定性研究论文撰写提示词.txt`
   - 可以根据需要调整这些提示词以获得更好的生成效果

### 启动应用

运行以下命令启动Web应用：

```bash
python app.py
```

然后在浏览器中访问：`http://localhost:5000`

### 使用流程

#### 步骤1：选择文献

1. 在Web界面中查看已识别的PDF文件列表
2. 使用"全选"按钮或手动选择要处理的文献
   - 注意：只有有引用信息的文献才能被处理
3. 点击"处理选中的PDF"按钮开始处理

#### 步骤2：文献总结

1. 系统会自动处理选中的文献
2. 界面会显示处理进度和预计剩余时间
3. 处理完成后，会显示每篇文献的总结结果（**注意仔细查看文献总结是否完整生成，如果有未生成的或者生成的质量不高、未生成完整等情况，可删除 `cache` 文件夹中的 `summaries_cache.json` 文件，重新运行了；也可以针对性的删除回复质量不高的文献，然后重新运行，已经生成总结的文献不会再次生成，除非删除 `summaries_cache.json` 中对应的部分**）
4. 点击"继续生成最终文稿"进入下一步

#### 步骤3：生成文稿

1. 选择要生成的文稿类型：
   - 文献综述(Literature Review)
   - 定性研究论文
2. 点击"生成最终文稿"按钮
3. 系统生成完毕后，会显示生成的文稿内容
4. 可以使用"下载文稿"按钮保存结果

## 高级功能与技巧

### 缓存管理

- 系统默认启用缓存功能，避免重复处理相同文献
- 如需重新处理已缓存的文献，可以：
  - 在 config.yaml 中设置 `cache.enabled` 为 `false`
  - 或删除 `cache/summaries_cache.txt` 文件

### 自定义提示词

提示词文件存放在 TXT_Files 目录中，可以根据需要修改以优化生成效果：

- **文献总结提示词**：控制单篇文献总结的内容和结构
- **文献综述撰写提示词**：控制最终文献综述的格式和风格
- **定性研究论文撰写提示词**：控制定性研究论文的结构和内容

### 调整模型参数

在 config.yaml 的 `llm` 部分可以调整以下参数：

- `temperature`：控制生成内容的创造性和多样性（较高值产生更多样化内容）
- `model_name`：选择不同的Gemini模型
- `request_timeout`：处理复杂文档可能需要延长超时时间

## 常见问题排解

1. **无法加载PDF文件**
   - 检查文件路径是否正确
   - 确认PDF文件格式有效且未加密

2. **引用匹配失败**
   - 检查 `参考文献列表.txt` 中的引用格式
   - 确保引用信息中包含的作者、年份等信息能与PDF文件名对应

3. **生成总结失败**
   - 检查API密钥是否有效
   - 查看网络连接是否稳定
   - 检查PDF内容是否过大导致超出模型上下文限制

4. **最终文稿生成失败**
   - 确保有足够的成功总结作为输入
   - 尝试调低temperature值以获得更确定性的输出
   - 检查 `文献综述撰写提示词.txt` 或 `定性研究论文撰写提示词.txt` 是否正确

5. **界面显示错误**
   - 查看浏览器控制台是否有JavaScript错误
   - 确保使用最新版本的现代浏览器

## 注意事项

- 处理大量PDF文件可能会导致较长的处理时间
- API调用可能会产生费用，请关注您的API使用额度

- 所有中文文件名需要正确保存，避免编码问题
